v1.1 storyboard · repo-brain · agent-init · all-cloud

init from your coding agent.
A teammate joins without ever touching git.

The happy path, scene by scene: a developer runs init from Claude Code / Codex inside their repo, the agent triages which docs move to a repo-linked cloud folder and which stay in git, the desktop app picks up syncing — then a non-technical, agent-native teammate joins zero-install and the two co-create in real time. One cloud authority, one universal bridge, nothing in git.

Workspace

Your top container — where membership lives. Holds folders.

Folders (nest & share)

Organize docs, link to a repo, and share with collaborators. A "brain" is just a repo-linked folder.

Cloud-only docs

Every doc lives in the cloud CRDT. Never in git → deletable, revocable, no permanent history to leak.

Two surfaces, one entry

Web = zero-install human editing. Desktop app = the local bridge for agents. Entry = an init skill run from Claude Code / Codex.

2

Triage — what moves, what stays

The agent walks the dev through a split: opinion, strategy, evolving prose — anything a non-dev collaborator needs — moves to the cloud folder. Code-coupled truth — README, CLAUDE.md, ADRs, anything the codebase needs at clone time — stays in git. The dev edits the list and confirms.

✓ brain/q3-narrative.md — strategy · collaborative
✓ notes/pricing-bets.md — sensitive · should be revocable
✓ specs/architecture.md — living doc · agents read it

· README.md — the repo's front door
· CLAUDE.md — agent config, belongs to the clone
· docs/adr/ — engineering record, wants line diffs
› confirm? [enter = apply · e = edit list]
⚖️

The heuristic: if it's opinion that evolves and someone without repo access needs it → cloud. If the codebase itself needs it at clone/build time → git.

🕰️

Honest scope: files already committed keep their git history — moving stops future history, it doesn't erase the past. Brand-new docs get the full revocability guarantee from day one.

4

Why nothing is in git

All-cloud isn't a limitation — it's the privacy guarantee. Anything in git has permanent history that can't be revoked. Keeping everything cloud-only makes "un-share and delete" behave like a Google Doc, with no asterisk.

absolute revocability
In git
history is forever · leaks on clone · merge conflicts
✕
Cloud-only
deletable · re-permissionable · CRDT merges, no conflicts
📄

Agents still get files — docs project to a gitignored synced folder on disk. On-disk ≠ in-git. Version history & comments live in the cloud, richer than line-diffs.

🧭

Later, optional: a folder-level "sync back to the repo" export, for the few docs that genuinely belong in the codebase. Not v1.

9

The one bridge: the watcher reconciles

The desktop app watches the filesystem, not any app. So a Cowork save and a TextEdit save are identical to it: diff against the base → scoped patch → merge into the cloud CRDT. This bridge is universal and can never be retired.
